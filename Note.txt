异常笔记

1 异常的主要功能是对程序的错误控制
可以分为两个方面：异常的抛出 以及异常的检测

关于异常的检测
可以说的是程序中会有很多不同层次，不同类型的异常，只要你调用其他人的方法，这些方法对于你来说是不可知的（说的
是你不知道它的实现机制），你不能保证它能够正确执行。这时就需要程序的异常处理机制了。
一般来说，程序的API说明中都会对方法抛出的异常进行说明。我们可根据这些说明进行异常的捕获并进行处理。
但是，不可能对每个函数调用语句或程序块都捕获异常，这就需要：
(1) 要明确程序中那些是绝对不会或不应该抛出异常的，那些是可能抛出异常的
  可能抛出异常的都有一些未知因素，比如网络等。所有一般都会在连接数据库，IO读写，网络连接时捕获异常
  try
  {
    IDBConnection con = ...
  }
  finally
  {
    con.Close();
  }

这是说错误破坏了程序功能的完整性或数据完整性，继续运行下去是错误的。

关于异常类的设计
异常类的层次结构应该“矮而宽”，即类层次要少，并且通过类名就可以知道该异常所代表的含义
